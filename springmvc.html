Spring Boot MVC

Spring Boot позволяет быстро и легко настаивать Spring приложения, имеет встроенный веб-сервер (Tomcat или Jetty), имеет конфигурацию на аннотациях (без xml-файлов).

MVC

MVC – архитектура приложения, при котором оно разбивается на три части Model, View, Contoller.С точки зрения Java, это три группы классов которые имеют слабые связи между собой, но сильные внутри себя.

  • Model - содержит в себе бизнес логику приложения.
  • View - содержит в себе все что связано с отображением данных пользователю.
  • Controller - занимается обработкой данных пользователя.

Создаем проект

Актуальный гайд лежит на оф. сайте
1. Создаем простой Maven проект и добавляем в него необходимые зависимости и плагины для веб.

Создаем контроллер

Контроллер нужен для того чтобы возвращать html-страницы которые мы замапили на какой то адрес.
Создаем src/main/java/hello/GreetingController.java

Создаем страницу

В папке temlate, в ресурсах создаем первую из многих страницу, которы будет отдавать в ответ на запросы. src/main/resources/templates/greeting.mustache

Создаем класс запуска

Создаем точку входа в наше Spring-приложение src/main/java/hello/GreetingController.java

Теперь при запуске на localhost развернется сервер Tomcat, который по умолчанию будет слушать порт 8080. На сервере запустится наше приложение и будет отдавать html страницу в ответ на запрос который мы замапили, а именно localhost:8080/greeting.

Если возникают какие то сложности по запуску на данном порту. Нужно добавить в папку resources файл application.properties и записать в него на каком порту запускаться. server.port=8083
