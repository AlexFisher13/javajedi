Базы данных

Любые данные необходимо каким то образом хранить, обрабатывать и извлекать. Информатика не была бы информатикой если бы не работала с информацией. Разного вида информацию удобно хранить в различных хранилищах иди базах данных.

Иерархические БД

В такой БД используется представление данных в виде древовидной структуры, состоящей из объектов различных уровней. Между объектами существуют связи, каждый объект может включать в себя несколько объектов более низкого уровня. Такие объекты находятся в отношении предка к потомку. Классический пример иерархической БД является файловая система состоящая из корневого каталога, в котором имеется иерархия подкаталогов и файлов.

Документ-ориентированые БД

Такая модель данных позволяет хранить информацию об объектах в виде ключ-значение, а именно в формате JSON. Каждый такой объект имеет вложенную структуру и объединяются в коллекции. Данный формат храннения данных позволяет очень удобно серилизовывать объекты из объектно-ориентированных языков программирования. Например сохранять java-объект в формате JSON очень просто. Примеры таких БД: MongoDB, CouchDB, Couchbase, MarkLogic, eXist, Berkeley DB

Реляционные БД

Реляционная модель данных хранит информацию в таблицах, а таблицы в свою очередь имеет отношения между собой. Данные о пользователе в одной таблице, с помощью уникального ключа относятся к другой таблице, например с адресами. Одна и та же информация не должна храниться в нескольких таблицах, это называется нормализацией. Обработка и выброка информации в такой БД основана на реляционной алгебре. Таким образом для работы с данным был создан язык SQL (structured query language) — язык структурированных запросов. Примеры таких БД: Oracle, MySQL, PostgreSQL.

ORM

Object-relation mapping. Реляционные БД продолжительное время, да и на данный момент являются самыми популярными БД. Хотя в ООП языках программирования перенос объектов в реляционный формат вызывал большие трудности. В связи с этим и появилась ORM - технология помогает связывать ООП объекты с реляционной моделью данных. Самой популярной ORM на сегодняшний день является Hibernate.
