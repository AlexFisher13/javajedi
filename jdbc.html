JDBC

JDBC (Java DataBase Connectivity) - стандарт взаимодействия Java-приложений с различными СУБД. JDBC представляет собой набор драйверов для разных СУБД. Для начала работы в приложении нужно получить экземпляр драйвер и открыть connection к БД, указав ее URL, логин и пароль. После чего можно выполнять запросы.

JDBC по сути представляет собой обычно 2 jar архива. Один jar набор интерфейсов для работы с БД, а второй это драйвер для конкретной БД, который представляет собой свою собственную реализацию этих интерфейсов для каждой конкретной БД.

JPA

JPA (Java Persistance API) – спецификация, в которой описаны основные правила для реализации принципов в ORM для Java, посредством JDBC.
JPA имеет множество реализаций:

  • OpenJPA (Apache)
  • TopLink (Oracle)
  • Hibernate (самый популярный, которые используется также в Spring JPA)

Hibernate

Hibernate – это ORM фреймворк для Java, реализация JPA спецификации. Эта технология является крайне мощной и имеет высокие показатели производительности.
Hibernate создаёт связь между таблицами в базе данных (далее – БД) и Java-классами. Это избавляет разработчиков от огромного количества лишней, рутинной работы, в которой крайне легко допустить ошибку и крайне трудно потом её найти.

Преимущества Hibernate:

  • Обеспечивает простой API для записи и получения Java-объектов в/из БД.
  • Минимизирует доступ к БД, используя стратегии fetching.
  • Не требует сервера приложения.
  • Позволяет нам не работать с типами данных языка SQL, а иметь дело с привычыми нам типами данных Java.
  • Заботится о создании связей между Java-классами и таблицами БД с помощью XML-файлов не внося изменения в программный код.
  • Hibernate поддердивает все основные СУБД: MySQL, Oracle, PostgreSQL, Microsoft SQL Server Database, HSQL, DB2.

Аннотации

В Hibernate предусмотрена возможность конфигурирования прилоения с помощью аннотаций.
@Entity указывает Hibernate, что данный класс является сущностью (entity bean). Такой класс должен иметь конструктор по-умолчанию (пустой конструктор).
@Table указывает Hibernate, с какой именно таблицей необходимо связать (map) данный класс. Аннотация @Table имеет различные аттрибуты, с помощью которых мы можем указать имя таблицы, каталог, БД и уникальность столбцов в таблиц БД.
@Id указываем первичный ключ (Primary Key) данного класса.
@GeneratedValue используется вместе с аннотацией @Id и определяет такие паметры, как strategy и generator.
@Column(name=”user”) определяет к какому столбцу в таблице БД относится конкретное поле класса (аттрибут класса)
