Веб-серверы и сервлеты

Раньше разработка любого веб-приложения на java сопровождалась написанием огромного количества одинакового кода, который выполнял прием и обработку веб-запросов.

JCP(JavaCommunityProcess) решили стандиртизовать этот код и вынести наружу, и назвать это веб-сервисом, задача которого будет приём и обработка веб-запросов и преобразование их в объекты для дальнейшей удобной работы с ними.
И теперь вместо того чтобы писать веб-приложение внутри которого будет обработчик веб-запросов, разработчик своё java приложение встраивает внутрь веб-сервера. На данный момент времени 99% java приложений не запускаются самостоятельно, они работают внутри веб-сервера.

Также JCP(JavaCommunityProcess) разработала такой стандарт как Servlet. По сути это java-класс, которому веб-сервер передаёт уже обработанный запрос для дальнейшей работы.

DispatcherServlet в Spring - это главный контроллер в приложении Spring MVC, который обрабатывает все входящие запросы и передает их для обработки в различные методы в контроллеры (@Controller, @RestController).

Так как веб-сервисы реализовывают работу с сервлетам их называют сервлет-конетйнерами. Самые популярные: Tomcat, Jetty, JBoss, GlassFish.
Также есть очень тяжеловестные и мощыне интерпрайз решения - Weblogic(Oracle) и WebSphere(IBM). Их функционал не огрничивается обработкой запросов, они помимо сервлет контйнеров поддерживоют огромное кол-во функионала. Например, главной их особенностью является свободная поддержка кластеров - это когда одно приложение запускается на множестве физических компьютеров, на них всех установлен один и тот же веблоджик например, и она автоматически балансирует нагрузку. Программист просто пишет код не заботясь обо всем этом, а веб-сервер сам всё делает.

Реализация веб-сервисов в Java: JAVA SERVLETS

Java Servlets – это технология реализующая возможности веб-сервиса, то есть она позволяет принимать, обрабатывать сетевые запросы и отправлять ответы.

Фактически в java это класс, расширяющий клаcс HttpServlet, и помеченный аннотацией @WebServlet, у которого есть два главных метода:
void doGet(HttpServletRequest request, HttpServletResponse response) {}
void doPost(HttpServletRequest request, HttpServletResponse response) {}

По сути сервлет связывает ссылку из адресной стокрои и java-класс, которые выполняет какую то логику и возвращает отвтет в виде HTML, JSP, JSON и др.
